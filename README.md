# Transformer Models for Edit Distance and Error Correction Codes

**Overview**
This repository holds a PyTorch study of how well Transformer models handle error correction code sequences, scored against traditional algorithms such as the BCJR decoder. The question runs in two parts. Can a model track the edit distance between sequences, and can it recover the original bits of a corrupted codeword. The work passed through three architectures, and each one answered a different half of that question.

**Phase One: Generative Sequence-to-Sequence**
The first model was an autoregressive sequence-to-sequence Transformer trained with a positional binary cross-entropy loss. Because it prints bits, it could be scored on every metric. It reached a Pearson correlation of 0.8589 and a Spearman correlation of 0.8557 when its generated sequences were compared to the targets by edit distance, with an RMSE of 2.4817 and an MAE of 1.9312. It also produced a Bit Error Rate of 0.3716 and a Hamming Loss of 0.3716. The failure was the Block Error Rate of 0.9999. A single deletion shifts every later bit, and the positional loss reads that shift as a full failure, so almost no block came out clean. The plots for this phase are in the seq directory and the older PDF reports under docs.

**Phase Two: Siamese Transformer**
To remove the shift penalty, the decoder was dropped and the encoder was reused as a Siamese network that maps each sequence into a 256-dimensional space. Sequences are compared by Euclidean distance, which avoids alignment. The encoder uses an embedding dimension of 256, eight attention heads, and four layers, with mean pooling to one vector per sequence. Training used Triplet Margin Loss over anchor, positive, and negative sequences for 30 epochs with AdamW. This phase reached the lowest absolute error of any learned model tested, an RMSE of 2.092 and an MAE of 1.670, with a Triplet Accuracy of 0.7238. Because it outputs a distance rather than bits, Hamming Loss and Bit Error Rate do not apply. The plots for this phase are in the plots directory.

**Phase Three: Connectionist Temporal Classification**
The third attempt added a Connectionist Temporal Classification head to recover bit sequences again while keeping the encoder-only design. The encoder output is upsampled by a factor of two with nearest-neighbor interpolation so the alignment lattice has room for the blank symbols the loss requires between repeated bits. This model did not converge. On the full one million sequence set the loss settled near 0.686, close to the natural logarithm of two, and the model collapsed to predicting the bit marginal. It scored a Bit Error Rate of 0.5859, worse than copying the input, which scores 0.0823. An overfitting test on 128 samples could not drive the error down either, which points at the nearest-neighbor upsampling rather than the learning rate. A learned upsampling layer is the planned fix. The training and evaluation logs and the error plots are in the ctc directory.

**Repository Layout**
The src directory holds the Python modules. The dataset directory holds the local HDF5 sequence arrays, which are not tracked in git. The plots directory holds the Siamese figures, the seq directory holds the generative figures, and the ctc directory holds the Connectionist Temporal Classification logs and figures. The docs directory holds the LaTeX report, the older PDF reports, and ctc/history.txt, which records the phase-by-phase metrics.

**Current Source Modules**
The active code in src is the Connectionist Temporal Classification pipeline and its diagnostics. model.py defines the CTCTransformer, the positional encoding, and the temporal upsampling. dataset_ctc.py loads the edit distance file and builds padded inputs and targets with the symbol offset that reserves index zero for the blank. train_ctc.py runs the production training loop on a CUDA device with per-sample input lengths. eval_ctc.py decodes the trained model and reports Bit Error Rate, Block Error Rate, and Hamming Loss. ctc_decode.py implements greedy decoding and the length-aware Levenshtein alignment behind those metrics. test_ctc.py, train_ctc_verify.py, train_ctc_overfit.py, diagnose_ctc.py, and diagnose_lengths.py are the verification and diagnosis scripts used to characterize the collapse and the channel.

**Earlier Source Modules**
The generative and Siamese code is preserved in the git history, and several evaluation scripts remain in src. eval.py, eval_metrics.py, eval_advance.py, and pca_analysis.py compute the triplet accuracy, the distance correlations, and the encoder Principal Component Analysis from the earlier phases. These scripts depend on model definitions from those phases and on the trained weight files, neither of which ships in the current working tree.

**Running the Code**
All scripts run from the repository root so the relative dataset path resolves. To verify the decoder and the alignment mathematics on synthetic input, run python src/ctc_decode.py. To check the channel statistics, run python src/diagnose_lengths.py and python src/diagnose_ctc.py. To train the Connectionist Temporal Classification model on a CUDA device, run python src/train_ctc.py, and to score it, run python src/eval_ctc.py.
